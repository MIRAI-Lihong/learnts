# 进阶

## 类型别名

```ts
type Name = string
type NameResolver = () => string
type NameOrResolver = Name | NameResolver // 常用于联合类型
function getName(n: NameOrResolver): Name {
  if (typeof n === 'string') {
    return n
  } else {
    return n()
  }
}
```

## 枚举

### 基本语法

```ts
// 常数项
enum Days {
  Sun,
  Mon,
  Tue,
  Wed,
  Thu,
  Fri,
  Sat
}

// 默认会被赋值为从 0 递增的数字，可以反向映射
Days['Sun'] === 0 // true
Days[0] === 'Sun' // true

// 手动赋值
enum Days {
  Sun = 7,
  Mon = 1,
  Tue,
  Wed,
  Thu,
  Fri,
  Sat
}
// 计算所得项
enum Color {
  Red,
  Green,
  Blue = 'blue'.length // 计算所得项
}
```

> 如果紧接在计算所得项**后面**的是**未手动赋值的项**，那么它就会因为无法获得初始值而报错

**常数项条件：**

- **不具有初始化**函数并且之前的枚举成员是**常数**
- 枚举成员使用**常数枚举表达式初始化**
  - **数字**字面量
  - 引用**之前定义的常数**枚举成员（可以是在**不同的**枚举类型中定义的）
  - 带括号的常数枚举表达式
  - +, -, ~ 一元运算符应用于常数枚举表达式
  - +, -, \*, /, %, <<, >>, >>>, &, |, ^ 二元运算符，常数枚举表达式做为其一个操作对象（结果为 NaN 或 Infinity 会报错）

### 常数枚举

使用`const enum`定义的枚举类型

```ts
const enum Directions {
  Up,
  Down,
  Left,
  Right
}

let directions = [
  Directions.Up,
  Directions.Down,
  Directions.Left,
  Directions.Right
]
// [0 /* Up */, 1 /* Down */, 2 /* Left */, 3 /* Right */]
```

> 会在编译阶段被删除，并且不能包含计算成员

### 外部枚举

```ts
declare enum Directions {
  Up,
  Down,
  Left,
  Right
}

let directions = [
  Directions.Up,
  Directions.Down,
  Directions.Left,
  Directions.Right
]
```

> declare 定义的类型只会用于编译时的检查，编译结果中会被删除

## ts 中的 class

### 修饰符

- public 修饰的属性或方法是公有的，默认权限
- private 修饰的属性或方法是私有的，不能在声明它的类的外部访问
- protected 修饰的属性或方法是受保护的，它和 private 类似，区别是它在子类中也是允许被访问的

### readonly

```ts
class Animal {
  // public readonly name;
  public constructor(public readonly name) {
    // this.name = name;
  }
}
```

> 修饰符和readonly还可以使用在构造函数参数中，等同于类中定义该属性同时给该属性赋值

### 抽象类

`abstract` 用于定义抽象类和其中的抽象方法

```ts
// 定义抽象类
abstract class Animal {
  public name
  public constructor(name) {
    this.name = name
  }
  public abstract sayHi() // 抽象方法
}
```

- 抽象类是**不允许被实例化**
- 抽象类中的抽象方法**必须被子类实现**

## 泛型

### 语法

在函数名后添加了 `<T>`，其中 T 用来指代任意输入的类型，在后面的输入 `value: T` 和输出 `Array<T>` 中即可使用

```ts
function createArray<T>(length: number, value: T): Array<T> {
  let result: T[] = []
  for (let i = 0; i < length; i++) {
    result[i] = value
  }
  return result
}

createArray(3, 'x') // ['x', 'x', 'x']

// 可以定义多个
function swap<T, U>(tuple: [T, U]): [U, T] {
  return [tuple[1], tuple[0]]
}
```

### 泛型约束

```ts
interface Lengthwise {
  length: number
}

// 只允许函数传入包含 length 属性的变量
function loggingIdentity<T extends Lengthwise>(arg: T): T {
  console.log(arg.length)
  return arg
}
```

### 泛型接口

```ts
interface CreateArrayFunc<T> {
  (length: number, value: T): Array<T>
}
```

### 泛型类

```ts
class GenericNumber<T> {
  zeroValue: T
  add: (x: T, y: T) => T
}
```

### 泛型参数的默认类型

```ts
function createArray<T = string>(length: number, value: T): Array<T> {
  let result: T[] = []
  for (let i = 0; i < length; i++) {
    result[i] = value
  }
  return result
}
```

## 装饰器

### 什么是装饰器

一种允许动态地往一个对象上添加自定义行为，而又不影响该对象所属的类的其他实例的一种设计模式

### 使用场景

- **复用逻辑**：避免在多个方法中写重复代码，将公共逻辑抽成装饰器
- **依赖注入**：结合 `@injectable` / `@inject` 管理类之间的依赖关系
- **AOP 面向切面**：在不修改原方法的前提下，插入日志、性能监控等逻辑

```ts
// 方法装饰器示例：记录执行时间
class MyService {
  @recordExecution
  myFn() {
    // do something...
  }
}

function recordExecution(
  originalMethod: any,
  context: ClassMethodDecoratorContext
) {
  return function (...args: any[]) {
    console.time('mark execution')
    originalMethod.call(this, ...args)
    console.timeEnd('mark execution')
  }
}
```

### 装饰器的类别

可以用来装饰四种类型的值：**类**、**类的属性**、**类的方法**、**属性存取器（getter/setter/accessor）**

所有装饰器的函数签名都是一致的，接收两个参数：

- `value`：被装饰的对象（方法本身、类本身等）
- `context`：存储上下文信息的对象

```ts
function myDecorator(value: any, context: DecoratorContext) {
  // ...
}
```

**执行顺序：**

1. 计算各个装饰器的值，按照从左到右，从上到下的顺序。
2. 调用方法装饰器。
3. 调用类装饰器。

### context 对象

所有装饰器的 context 均包含以下字段：

- `kind`：被装饰值的类型，可取 `class` / `method` / `field` / `getter` / `setter` / `accessor`
- `name`：被装饰值的名字
- `addInitializer`：接收一个回调，侵入初始化过程，常用于自动绑定 `this`
- `metadata`：在运行时访问编译时写入的元数据（通过 `Symbol.metadata` 读取）

除类装饰器外，其他装饰器的 context 还包含：

- `static`：是否被 `static` 修饰
- `private`：是否被 `private` 修饰
- `access`：运行时访问 value 的对象，提供 `has` / `get` / `set` 方法

```ts
const typeToYellingMap = {
  cat: 'meow~ meow~'
}

let yellingMethodContext: ClassMethodDecoratorContext

class Animal {
  type: string
  constructor(type: string) {
    this.type = type
  }

  @yelling
  greet() {
    console.log(`Hello, I'm a(n) ${this.type}!`)
  }

  accessor y = 1
}

function yelling(originalMethod: any, context: ClassMethodDecoratorContext) {
  yellingMethodContext = context
  return function (this: any, ...args: any[]) {
    console.log(typeToYellingMap[this.type as keyof typeof typeToYellingMap])
    originalMethod.call(this, ...args)
  }
}

const xcat = new Animal('cat')
xcat.greet() // meow~ meow~
// Hello, I'm a(n) cat!
yellingMethodContext.access.get(xcat).call(xcat) // meow~ meow~
// Hello, I'm a(n) cat!
console.log(yellingMethodContext.access.has(xcat)) // true
```
